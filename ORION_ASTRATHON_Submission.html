1. Problem Statement Chosen

Autonomous 3-D Asteroid Surface Shape Reconstruction from Multimodal Partial Observations. Build a fully automated pipeline that ingests sparse multimodal observational data (optical light curves and delay-Doppler radar images) and produces a high-fidelity, watertight 3-D mesh of an asteroid's surface, benchmarked against spacecraft-derived ground-truth models.

2. Problem Understanding

Only ~300 of 1,000,000+ discovered asteroids have rigorous 3-D shape models. Current methods (SHAPE suite) require months of expert intervention and cannot scale to the growing NEO catalogue. Accurate shapes are critical for planetary defence, Yarkovsky/YORP effect modelling, and physical characterisation (volume, density, albedo).

The Complementary Data Problem

Modality Strength Limitation
Delay-Doppler Radar Sub-10m resolution cross-sections North-South ambiguity (3D → 2D)
Optical Light Curves Precise spin axis + period Blind to concave features

Key insight: Neither modality alone suffices. Radar resolves fine detail but loses a dimension; light curves constrain global shape but only the convex hull. Multimodal fusion compensates for both blind spots.

Technical Constraints

  1. Full automation — no human-in-the-loop after configuration
  2. Non-convex recovery — craters, boulder fields, bifurcated lobes
  3. Watertight output — closed mesh exportable as .obj/.stl
  4. Physical correctness — radar↔body frame transforms must be sound

3. Implementation Roadmap

Phase Hours Tasks
1. Data Pipeline 0–8 Download Bennu/Eros ground-truth from NASA PDS; fetch light curves from DAMIT; acquire radar matrices from JPL; preprocessing (normalisation, phase-folding); build synthetic training data generator
2. Model Architecture 8–24 Dual-stream encoder (1D CNN+BiLSTM for light curves, ResNet-50 for radar); attention-based fusion module; SPHARM coefficient predictor; spherical harmonic mesh decoder; composite loss function
3. Training 24–36 Train on synthetic data with augmentation; fine-tune on real observations; optimise SPHARM degree; automated hyperparameter selection
4. Evaluation 36–48 Compute 5 metrics vs ground truth; side-by-side 3D renderings; export .obj meshes; metric report notebook; final presentation

5. Architecture

AsteroMesh — Dual-Stream Multimodal Fusion Network

Pipeline Overview

  1. Light-Curve Encoder: 1D CNN extracts local flux features → BiLSTM captures rotational-phase dependencies → 256-dim vector
  2. Radar Encoder: ResNet-50 processes delay-Doppler matrices → 256-dim spatial feature vector
  3. Attention Fusion: Cross-modal attention learns modality weighting → 512-dim fused representation
  4. SPHARM Predictor: FC layers → {alm, blm} coefficients up to degree l=25 (~676 coefficients)
  5. Mesh Decoder: r(θ,φ) = Σ(almcos + blmsin)Plm on spherical grid → triangulate → watertight genus-0 mesh

Key Design Decisions

Decision Rationale
SPHARM over direct vertex prediction Guarantees watertight, genus-0 topology by construction
Attention fusion over concatenation Adaptive modality weighting per surface region
Composite loss (Chamfer + Hausdorff + IoU) Balances average accuracy, worst-case error, and volume fidelity
